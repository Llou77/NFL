How It Works

This predictor uses a stacked machine learning ensemble trained on the last 3 NFL seasons (816 regular season + 39 playoff games) to forecast the final score of every upcoming game. The model outputs a predicted score for each team — from which spread, total, and moneyline predictions are derived automatically.

The pipeline runs automatically every Tuesday via GitHub Actions, fetching updated statistics and retraining the model on a rolling window that always covers exactly the 3 most recent seasons.

Feature Groups

The model uses ~150 input features per game, grouped into the following categories:

Team Rolling Statistics

Computed over the last 4, 8, and 16 games played by each team (strictly prior to the predicted game):

EPA/play offense EPA/play defense Success rate Pass EPA Rush EPA CPOE Air yards/att 3rd down rate Red zone TD% Turnover diff Sack rate Pressure rate FG% Shotgun rate Win rate Score differential Explosive play rate

Matchup Gap Features

For each game, the offensive EPA of Team A is compared directly against Team B's defensive EPA allowed, creating a head-to-head gap feature. This is done for passing, rushing, 3rd down, and red zone efficiency.

Contextual Features

Home/Away flag Rest days Rest differential Bye week flag Short week (TNF) Temperature Wind speed Dome/Outdoor Primetime flag International game Division rivalry Week number Referee tendencies

Head-to-Head History

Historical matchup records between each pair of teams, with a tiered lookback: 3 seasons for division rivals, 6 seasons for conference opponents, 10 seasons for cross-conference matchups. H2H features are modulated by roster continuity — if both teams have largely different personnel, the historical H2H record is discounted.

Injury & Roster

QB availability Offensive availability score Defensive availability score Injury report severity O-line health

Cross-Season Signals

Roster continuity Vegas preseason wins O/U Draft quality score Elo rating Elo vs. spread divergence QB change flag Coach change flag

Model Architecture

A 3-layer stacked ensemble where each layer's predictions feed into the next:

Layer 1A
Ridge Regression
Linear baseline. Interpretable, handles multicollinearity. Trained with L2 regularization.
Layer 1B
XGBoost
Gradient boosted trees. Captures non-linear interactions. MAE objective.
Layer 1C
LightGBM
Faster gradient boosting alternative. Leaf-wise growth strategy.
Layer 2
Dual-Head Neural Network
PyTorch NN (256→128→64) with separate heads for home and away scores. Multi-target loss optimizes score, total, and spread simultaneously.
Layer 3
Meta-Learner
Ridge regression that learns the optimal blend of all sub-model outputs. Trained on out-of-fold predictions to prevent overfitting.

Confidence Score

Every prediction includes a confidence score (0–100%) composed of four weighted components:

Model Agreement (40%)

How closely the Ridge, XGBoost, LightGBM, and Neural Network predictions agree. High std-dev between models → lower confidence.

Feature Completeness (30%)

What fraction of expected features are non-null. High-importance features (QB EPA, Elo) carry extra weight when missing.

H2H Data Quality (15%)

Sample size of available head-to-head historical games. Division games (6 in 3 years) score near 1.0; cross-conference games with 0 meetings score 0.2.

Injury Freshness (15%)

How current the injury report is relative to kickoff. Predictions generated <24h before kickoff score 1.0; >7 days out score 0.4.

Edge Suppression: When confidence is WEAK (<50%), edge indicators are hidden to prevent misleading signals driven by data gaps.

Sliding Window

The model trains on exactly 3 × 272 = 816 regular season games plus 39 playoff games. Each week after a game completes, the newest game enters the window and the oldest drops out. Season weights are optimised by Bayesian search (Optuna) rather than being fixed — ensuring recent games contribute appropriately more than older ones.
