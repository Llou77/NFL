Methodology

How the model is built, what data it uses, and how it is validated.

Overview

How the model is run, what data it trains on, and how it updates automatically.

Every Tuesday during the NFL season, a GitHub Actions pipeline retrains a stacked ensemble on a rolling window of the last four completed seasons plus the season in progress (~1,100 labeled games) and publishes predictions for every upcoming game. Historical data for finished seasons is sealed into the repository once a year (the "frozen layer"), so weekly runs only download current-season files.

The model predicts a final score for both teams. Spread, total, and moneyline signals are derived from these two numbers. Season sample weights are auto-optimised each pre-season via Bayesian search (Optuna). The pipeline knows the 2021 schedule expansion (16 games/17 weeks → 17 games/18 weeks), so season-progress features mean the same thing in every era.

Model Architecture

Three stacked prediction layers: base models → neural network → final blender.

Layer 1A
Ridge Regression
Simple linear model with a complexity penalty to prevent overfitting. Fast and interpretable.
Layer 1B
XGBoost
500 decision trees optimised for minimum absolute error. Captures complex, non-linear patterns.
Layer 1C
LightGBM
A fast gradient boosting variant that grows trees leaf-by-leaf for higher accuracy.
Layer 2
Neural Network
Deep neural network with separate output branches for home and away scores. Simultaneously optimises score accuracy, margin, and combined total.
Layer 3
Meta-Learner
A final blending layer that learns the optimal mix of all sub-model outputs, trained on held-out data to prevent overfitting.

Confidence Score

Measures how much the model trusts its own prediction — not the probability of winning.

Every prediction carries a 0–100% score measuring model agreement and data completeness. Edge signals are suppressed for low-confidence games.

Training window and data freshness

The last four completed seasons plus the season in progress (~1,100 labeled games). Each completed game enters the window and the oldest season exits. Playoff games weighted at ~0.4–0.7×. Season weights optimised each pre-season by Bayesian search; closed-season data comes from the committed frozen layer, so a renamed or deleted upstream file can never break a run.
